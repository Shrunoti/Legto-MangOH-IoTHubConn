\subsection{Detailed Description}
\hyperlink{struct_content__t}{Content\+\_\+t} is the base class for all file content items, including \hyperlink{struct_token__t}{Token\+\_\+t}, and \hyperlink{struct_compound_item__t}{Compound\+Item\+\_\+t}.

The root of each parse tree is a \hyperlink{struct_def_file__t}{Def\+File\+\_\+t}. It has a pointer to the first token (\hyperlink{struct_token__t}{Token\+\_\+t}) that was parsed from it. It also has a list of top-\/level sections (\hyperlink{struct_compound_item__t}{Compound\+Item\+\_\+t}).

Each \hyperlink{struct_token__t}{Token\+\_\+t} keeps track of its exact text and the file, line, and column where it was found. As tokens are parsed from the file, they are linked together into a doubly-\/linked list. The \hyperlink{struct_def_file__t}{Def\+File\+\_\+t} has a pointer to the first \hyperlink{struct_token__t}{Token\+\_\+t} in the file.

Each \hyperlink{struct_compound_item__t}{Compound\+Item\+\_\+t} has a type indicating what kind of item it is. It also has pointers to its first and last \hyperlink{struct_token__t}{Token\+\_\+t} and a list of \hyperlink{struct_content__t}{Content\+\_\+t} items that are inside it. If it is a simple section or named item with only a single name or number inside it, there will only be one \hyperlink{struct_token__t}{Token\+\_\+t} pointer inside this list. If it is a more complex section or named item, there could be any number of \hyperlink{struct_content__t}{Content\+\_\+t} objects in this content list, and it could even be empty if there\textquotesingle{}s nothing but a pair of curly braces with nothing but whitespace or comments between them.
