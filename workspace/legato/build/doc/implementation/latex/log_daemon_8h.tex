\subsection{Detailed Description}
This file contains the log control protocol specification.

Log control messages have the following format\+:

\begin{DoxyVerb} Command ProcessName '/' ComponentName '/' CommandData
\end{DoxyVerb}



\begin{DoxyItemize}
\item Command = one character indicating what type of command it is.
\item Process\+Name = variable-\/length name, up to L\+I\+M\+I\+T\+\_\+\+M\+A\+X\+\_\+\+P\+R\+O\+C\+E\+S\+S\+\_\+\+N\+A\+M\+E\+\_\+\+L\+EN characters.
\item Component\+Name = variable-\/length name, up to L\+I\+M\+I\+T\+\_\+\+M\+A\+X\+\_\+\+C\+O\+M\+P\+O\+N\+E\+N\+T\+\_\+\+N\+A\+M\+E\+\_\+\+L\+EN characters.
\item Command\+Data = optional additional command data, depending on the type of command.
\end{DoxyItemize}

The Log Control Daemon advertises two services. One is for log clients to connect to. The other is for log control tools to connect to.

Log clients connect and send in their log session identification information in a \char`\"{}\+Register\char`\"{} message, which includes the process name, component name, and process ID. If the Log Control Daemon has non-\/default session control data available for that session, then it will send log control commands to the log client at that time. Also, the Log Control Daemon will use log control commands to update log clients when log control settings are changed by log control tools.

\begin{DoxyRefDesc}{Todo}
\item[\hyperlink{todo__todo000005}{Todo}]Change to use shared memory to control log sessions instead.\end{DoxyRefDesc}


Log tools connect and send in a log control command. The Log Control Daemon responds by sending printable strings to the log control tool. The log control tool simply prints these strings to stdout when it receives them. The Log Control Daemon closes the I\+PC session with the log control tool when it finishes processing the command. Response strings that contain error messages always start with a \char`\"{}$\ast$\char`\"{}.
