Detailed Description

Log Session objects are used to store the log session details for a single, active log session in a running process. Each of these objects is kept on a Running Process object's Log Session List. They are keyed by component name.

Log session. Stores log configuration for each registered component. The component names and filters are created and stored by the components' themselves but no one should be accesssing them but this le_log module.
