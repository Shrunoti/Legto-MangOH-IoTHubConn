Detailed Description

Change notification handler object structure. (aka "Handler objects")

Each one of these is used to keep track of a client's change notification handler function registration for a particular tree node. These are allocated from the Handler Pool and kept on a Node object's Handler List.

Handler object.

This stores the registration information for a handler function. They are allocated from the Handler Pool and are stored on a thread's Handler List.

Warning
These can be accessed by multiple threads, and are in both the Event List structure and the Per-Thread structure. Great care must be taken to prevent races when accessing these objects (use the Mutex).
Note
The lifecycle of these objects is such that once they have been created, only their list links can be changed, until they are deleted.
