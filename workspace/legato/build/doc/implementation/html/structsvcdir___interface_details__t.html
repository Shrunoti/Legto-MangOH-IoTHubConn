Detailed Description

Name of the Service Directory's "Server Socket", which is a named Unix domain sequenced-packet socket (AF_UNIX, SOCK_SEQPACKET) that servers connect to when they want to offer a service. Name of the Service Directory's "Client Socket", which is a named Unix domain sequenced-packet socket (AF_UNIX, SOCK_SEQPACKET) that clients connect to when they want to access a service. Interface details. Both client and server need to send this information to the Service Directory when they connect.
