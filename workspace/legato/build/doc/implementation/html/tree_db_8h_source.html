3 /**
4  * @file treeDb.h
5  *
6  * Interface to the low level tree DB structure.
7  *
8  * There are basically two types of tree:
9  * - Named trees
10  * - Shadow trees
11  *
12  * Both are accessed the same basic way:
13  * 1. Get a reference to the tree.
14  * 2. Get a reference to the root node of the tree.
15  * 3. Traverse the tree.
16  * 4. Read/write the node values and/or create/delete new child nodes.
17  *
18  * The shadow trees are used to support write transactions. When a write transaction is
19  * started for a given named tree, a shadow tree is created to "shadow" that named tree.
20  * All changes to the tree are only done through the shadow tree. When the write transaction
21  * is committed, the shadow tree is "merged" back into the named tree and then the shadow tree
22  * is deleted. To cancel a write transaction, just delete the shadow tree without merging
23  * it back into the named tree.
24  *
25  * Copyright (C) Sierra Wireless Inc.
26  *
27  */
