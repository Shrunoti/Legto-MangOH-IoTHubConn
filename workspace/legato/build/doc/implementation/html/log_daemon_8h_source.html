1 /** @file logDaemon.h
2  *
3  * This file contains the log control protocol specification.
4  *
5  * Log control messages have the following format:
6  *
7  * @verbatim
8  Command ProcessName '/' ComponentName '/' CommandData
9 @endverbatim
10  *
11  * - Command = one character indicating what type of command it is.
12  * - ProcessName = variable-length name, up to LIMIT_MAX_PROCESS_NAME_LEN characters.
13  * - ComponentName = variable-length name, up to LIMIT_MAX_COMPONENT_NAME_LEN characters.
14  * - CommandData = optional additional command data, depending on the type of command.
15  *
16  * The Log Control Daemon advertises two services. One is for log clients to connect to.
17  * The other is for log control tools to connect to.
18  *
19  * Log clients connect and send in their log session identification information in a
20  * "Register" message, which includes the process name, component name, and process ID.
21  * If the Log Control Daemon has non-default session control data available for that
22  * session, then it will send log control commands to the log client at that time. Also,
23  * the Log Control Daemon will use log control commands to update log clients when log
24  * control settings are changed by log control tools.
25  *
26  * @todo Change to use shared memory to control log sessions instead.
27  *
28  * Log tools connect and send in a log control command. The Log Control Daemon responds
29  * by sending printable strings to the log control tool. The log control tool simply prints
30  * these strings to stdout when it receives them. The Log Control Daemon closes the IPC
31  * session with the log control tool when it finishes processing the command.
32  * Response strings that contain error messages always start with a "*".
33  *
34  * Copyright (C) Sierra Wireless Inc.
35  */
