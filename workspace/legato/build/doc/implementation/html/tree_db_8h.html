Detailed Description

Interface to the low level tree DB structure.

There are basically two types of tree:

  • Named trees
  • Shadow trees

Both are accessed the same basic way:

  1. Get a reference to the tree.
  2. Get a reference to the root node of the tree.
  3. Traverse the tree.
  4. Read/write the node values and/or create/delete new child nodes.

The shadow trees are used to support write transactions. When a write transaction is started for a given named tree, a shadow tree is created to "shadow" that named tree. All changes to the tree are only done through the shadow tree. When the write transaction is committed, the shadow tree is "merged" back into the named tree and then the shadow tree is deleted. To cancel a write transaction, just delete the shadow tree without merging it back into the named tree.
