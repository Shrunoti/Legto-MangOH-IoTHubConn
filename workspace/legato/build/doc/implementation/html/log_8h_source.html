1 /** @file log.h
2  *
3  * Log module's intra-framework header file. This file exposes type definitions and function
4  * interfaces to other modules inside the framework implementation.
5  *
6  * The Log Control Daemon is a server process that everyone else connects to. The log control
7  * tool sends commands to the Log Control Daemon who validates them and keeps track of log
8  * settings that last longer than the lifetime of a given process. When another process opens
9  * a log session with the Log Control Daemon, the Daemon updates that process with any settings
10  * that were previously set for processes that have that name.
11  *
12  * In the current implementation, the Log Control Daemon sends settings to processes using
13  * the IPC session. These get applied by a message receive handler running in the process's
14  * main thread.
15  *
16  * @section log_future Future Enhancement
17  *
18  * In the future, the Log Control Daemon will write log settings (filter level and keyword
19  * enable/disable) directly into the client process's address space using shared memory.
20  * The shared memory file is created by the log client who passes the shared memory file
21  * fd to the Log Control Daemon over the @ref c_messaging when the client starts up.
22  *
23  * When a process starts, it must create its shared memory file and define the layout of the
24  * settings within it before it talks to the Log Control Daemon. Once it has notified the
25  * Log Control Daemon of the file's location and layout, the size and layout of the file must
26  * not change for the lifetime of the process.
27  *
28  * The shared memory file layout is a list of log sessions, identified by component name.
29  * For each session, there exists a level, a set of output location flags, and a list of
30  * trace keywords, each of which has an "is enabled" flag.
31  *
32  * Copyright (C) Sierra Wireless Inc.
33  */
